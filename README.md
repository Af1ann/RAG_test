# RAG Demo

Демонстрационное приложение RAG (Retrieval-Augmented Generation) с использованием локальных моделей через Ollama.

## Возможности

- Загрузка документов форматов PDF и TXT
- Создание векторной базы данных на основе Chroma
- Поиск релевантных фрагментов по запросу
- Генерация ответов с использованием локальных LLM через Ollama
- Интерактивный CLI-интерфейс для вопросов

## Требования

- Python 3.8+
- [Ollama](https://ollama.ai/) с установленными моделями:
  - `gemma3:4b` (или другая модель для генерации)
  - `nomic-embed-text` (для эмбеддингов)

## Установка

1. Клонируйте репозиторий и перейдите в папку проекта:
   ```bash
   cd rag_demo
   ```

2. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv venv
   venv\Scripts\activate   # Windows
   # source venv/bin/activate   # Linux/macOS
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Убедитесь, что Ollama запущен:
   ```bash
   ollama serve
   ```

5. Скачайте необходимые модели (если ещё не скачаны):
   ```bash
   ollama pull gemma3:4b
   ollama pull nomic-embed-text
   ```

## Использование

1. Поместите ваши документы (`.pdf` или `.txt`) в папку `data/`

2. Запустите приложение:
   ```bash
   python rag_demo.py
   ```

3. Введите вопрос и получите ответ на основе загруженных документов

4. Для выхода введите `exit` или `quit`

## Конфигурация

Основные параметры можно изменить в начале файла `rag_demo.py`:

```python
DATA_DIR = "data"              # Папка с документами
PERSIST_DIR = "chroma_db"      # Папка для хранения векторной БД
MODEL_NAME = "gemma3:4b"       # Модель для генерации ответов
EMBEDDING_MODEL = "nomic-embed-text"  # Модель для эмбеддингов
```

## Структура проекта

```
rag_demo/
├── rag_demo.py       # Основной скрипт приложения
├── requirements.txt  # Зависимости Python
├── data/             # Папка для документов
│   └── test.txt      # Пример документа
└── chroma_db/        # Векторная база данных (создаётся автоматически)
```

## Как это работает

1. **Загрузка документов** — PDF и TXT файлы загружаются из папки `data/`
2. **Разбиение на чанки** — документы разбиваются на фрагменты по ~500 символов с перекрытием 50
3. **Создание эмбеддингов** — каждый фрагмент преобразуется в вектор с помощью `nomic-embed-text`
4. **Сохранение в Chroma** — векторы сохраняются в локальную базу данных
5. **Поиск и генерация** — при запросе находятся релевантные фрагменты и передаются в LLM для генерации ответа
